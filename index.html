elasticsearch-head 插件安装

发表于 2018-12-11

  • 这个插件对es5.0之后的版本极其不友好，查到的教程都有一些奇怪

  • 在线安装方法：

    • https://github.com/mobz/elasticsearch-head 下载elasticsearch-head-master.zip

      https://github.com/mobz/elasticsearch-head 下载 node-v10.14.1-linux-x64.tar.xz

    • 安装node.js

      • 解压

      • 配置环境变量 在 .bash_profile 中添加

        export NODE_HOME=/home/elasticsearch/node-v10.14.1-linux-x64
        export PATH=\$PATH:\$NODE_HOME/bin
        export NODE_PATH=\$NODE_HOME/lib/node_modules

        执行 source .bash_profile 然后node -v或者npm -v执行 查看node.js 版本

      • 安装完成

    • 安装head插件

      • 解压

      • 修改 Gruntfile.js文件 在

        connect: {
        ​ server: {
        ​ options: {
        ​ port: 9100,
        ​ base: ‘.’,
        ​ keepalive: true,
        ​ hostname: ‘*’ 添加这个字段,上面的逗号谁不加谁知道
        ​ }
        ​ }
        }

      • 修改 _site/app.js 文件 将

        this.base_uri=this.config.base_uri || this.prefs.get(“app-base_uri”) || “http://localhost:9200"; 中的localhost替换为当前机器的IP地址

      • 进入elasticsearch-head-master 目录 执行

        npm install

        npm install -g grunt-cli

        运行grunt server可能会报出下面的一些错误👇

        Local Npm module “grunt-contrib-clean” not found. Is it installed?
        Local Npm module “grunt-contrib-concat” not found. Is it installed?
        Local Npm module “grunt-contrib-watch” not found. Is it installed?
        Local Npm module “grunt-contrib-connect” not found. Is it installed?
        Local Npm module “grunt-contrib-copy” not found. Is it installed?
        Local Npm module “grunt-contrib-jasmine” not found. Is it installed?

        解决办法： 一个一个装上 举个例子如下 同理把6个都install（最后一个可能会报错 不影响）

        npm install grunt-contrib-clean --registry=https://registry.npm.taobao.org

      • 执行 grunt server 一般情况下就会正常执行了 访问 ip:9100即可

  • 离线安装方法：

    • 下载 elasticsearch-head-master.zip 和 node-v10.14.1-linux-x64.tar.xz，按照上一步操作到修改文件
    • 找一台能在线安装的机器，安装好之后拷贝出elasticsearch-head-master 文件夹下的 node_modules文件夹，放到离线机器的对应位置
    • 执行 node_modules/grunt/bin/grunt server即可

ES常见命令

发表于 2018-12-03

  • 创建索引: PUT http://192.168.115.81:9200/log?pretty

  • 删除索引: DELETE http://192.168.115.81:9200/twitter?pretty

  • 添加文档:

    POST http://192.168.115.81:9200/log/message/

    {
    ​ “raw”:”this is test5”,
    ​ “hostname”:”sy-test-5”,
    ​ “value”:”10000”,
    ​ “field_add_test”:7

    }

  • 删除文档:

    DELETE http://192.168.115.81:9200/log/message/OykHc2cBah8KRcK6RdIi?pretty

  • 更新文档:

    POST http://192.168.115.81:9200/log/message/mCk6eGcBah8KRcK6KNLS/_update

    {
    ​ “doc”:{
    ​ “raw1”: “this is c raw”
    ​ }
    }

  • 批量添加文档：

    POST http://192.168.115.81:9200/_bulk?pretty

    {“index”:{“_index”:”log”,”_type”:”message”}}\n
    {“name”: “sy” }
    {“index”:{“_index”:”log”,”_type”:”message”}}\n
    {“name”: “wx” }

  • 批量删除文档：(这里有一个比较骚的问题是 bulk方式在批量做的时候需要有换行 是真的需要换 不然会一直报error)

    POST http://192.168.115.81:9200/_bulk?pretty

    {“delete”:{“_id”:”1”}}\n
    {“delete”:{“_id”:”2”}}\n

ELK集群搭建记录

发表于 2018-11-22
  • es通过delete方法删除索引的时候，可以通过_all或者* 来全部删除，为了防止误删除，可以指定elasticsearc.yml文件中的 atino.destructive_requires-name为true 必须使用名称或者别名才能删除
  • kibana在linux下安装启动的时候需要修改host为当前机器的ip否则无法通过浏览器访问
  • es中，如果新添加的文档中含有之前未映射的字段，那么es会自动猜测该字段值的类型，安全的做法是在索引数据之前定义好所需的映射。 (已测试合理)

ES集群搭建

发表于 2018-11-22

Elasticsearch 6.5.1

  1. 下载

    地址 https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-6.5.1.tar.gz

  2. 修改配置文件 elasticsearch-6.5.1/config/elasticsearch.yml

    cluster.name: bizseer #集群名称

    path.data: /path/to/data #指定数据存储路径

    path.data: /path/to/data #指定日志存储路径

    network.host: ip_addr #指定当前机器ip

    http.port: 9200 #指定端口 默认为9200

    discovery.zen.ping.unicast.hosts: [host1，host2, …] #指定集群初始化时的节点

  3. 在三台机器上执行elasticsearch-6.5.1/bin/elasticsearch -d启动

可能出现的错误

若ps -ef | grep elasticsearch发现没有es进程，查看日志发现有报错信息如：

  • max virtual memory areas vm.max_map_count [65530] likely too low

    解决办法:

    su - root

    vi /etc/sysctl.conf

    在文件末尾添加 vm.max_map_count=262144

    sysctl -p

  • max file descriptors [4096] for elasticsearch process likely too low

    解决办法:

    su - root

    vi /etc/security/limits.conf

    在文件末尾添加

    * soft nofile 65536

    *hard nofile 131072

    * soft nproc 2048

    *hard nproc 4096

    切换为es用户 使用ulimit -n查看最大进程数是否修改了65536 若未修改 需要退出bash重新登录
